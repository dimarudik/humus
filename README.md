# Humus JDBC Proxy Wrapper

**Humus** — это легковесный JDBC-прокси драйвер на Java. Он позволяет расширять возможности существующего JDBC-драйвера PostgreSQL с помощью цепочки плагинов, не изменяя код основного приложения.

## 🚀 Основные возможности

- **Прозрачное проксирование**: Работает как стандартный JDBC-драйвер через префикс `jdbc:humus:`.
- **Архитектура плагинов**: Поддержка динамической загрузки плагинов через `ServiceLoader`.
- **Цепочка вызовов (Plugin Chain)**: Плагины выполняются последовательно, позволяя перехватывать запросы, изменять их или собирать метрики.
- **Динамический Discovery**: Возможность подмены целевого URL базы данных «на лету» (например, через gRPC сервис).
- **Диагностика и мониторинг**:
    - Логирование медленных запросов по настраиваемому порогу.
    - Автоматическое получение планов выполнения (EXPLAIN ANALYZE) для тяжелых запросов.

## 🛠 Установка и подключение

Библиотека подключается как стандартная зависимость. Для активации прокси необходимо изменить JDBC URL вашего приложения:

**Оригинальный URL:**
`jdbc:postgresql://localhost:5432/mydb`

**URL через Humus драйвер:**
`jdbc:humus:postgresql://localhost:5432/mydb`

## 🧩 Доступные плагины

### 1. ExecutionTimePlugin
Логирует время выполнения SQL-запросов.
- **Параметр:** `slow-query-threshold-ms`
- **Описание:** Если запрос выполняется дольше указанного времени, в лог выводится SQL-текст и время его выполнения (если параметр не установлен или равен 0, то плагин не активируется и логирование не происходит).

### 2. ExecutionPlanPlugin
Автоматически получает план выполнения для медленных запросов.
- **Параметр:** `slow-query-plan-threshold-ms`
- **Описание:** При превышении порога плагин выполняет обращение к PostgreSQL расширению для последнего успешного запроса и выводит результат в лог (если параметр не установлен или равен 0, то плагин не активируется и логирование не происходит).

### 3. DiscoveryPlugin (gRPC)
Позволяет динамически разрешать адрес базы данных через внешний сервис.
- **URL формат:** `jdbc:humus:grpc://discovery-service-host:9090/db_name`

[Подробнее](./src/main/java/sdk/humus/plugins/discovery/README.md)

## 🏗 Архитектура

Проект построен на нескольких ключевых компонентах:
- **`HumusDriver`**: Точка входа, регистрируется в `DriverManager`.
- **`ConnectionWrapper`**: Прокси-обертка над реальным соединением.
- **`ProxyPlugin`**: Интерфейс для реализации кастомной логики.
- **`ServiceLoader`**: Используется для автоматического обнаружения фабрик плагинов (`ProxyPluginFactory`).

## ⚙️ Требования
- **Java:** 11+.

## 🧪 Тестирование
Проект содержит интеграционные тесты с использованием **Testcontainers**. Для запуска тестов необходим Docker. Тесты проверяют:
- Корректность работы цепочки плагинов.
- Отсутствие дублирования вызовов (особенно для `UPDATE` запросов).
- Работу gRPC Discovery.
